# 数据字典

## 表一：raw_rent

| 字段名 | 类型 | 含义 | 示例值 |
| --- | --- | --- | --- |
| id | int | 自增主键 | 1 |
| house_id | varchar(50) | 原始数据源唯一房源 ID | CD2124753202778734592 |
| title | varchar(255) | 房源标题 | 锦江区卓锦城2室1厅 |
| district | varchar(50) | 原始抓取的行政区 | 锦江 |
| bizcircle | varchar(100) | 原始商圈或地址字段 | 卓锦城 |
| community | varchar(100) | 原始小区名称 | 鑫苑名家一期 |
| layout | varchar(50) | 原始户型描述 | 2室1厅1卫 |
| area | decimal(8,2) | 建筑面积（㎡） | 62.46 |
| price | int | 月租金（元） | 1850 |
| crawl_time | datetime | 抓取时间 | 2026-06-10 15:13:38 |

说明：`raw_rent` 为爬虫抓取后的原始房源数据，字段基本保持原始网页内容。

| 字段名 | 类型 | 含义 | 示例值 |
| --- | --- | --- | --- |
| id | int | 自增主键 | 1 |
| house_id | varchar(50) | 原始数据源唯一房源 ID | CD2124753202778734592 |
| district | varchar(50) | 标准化后的行政区 | 锦江区 |
| bizcircle | varchar(100) | 标准化后的商圈 | 卓锦城 |
| community | varchar(100) | 标准化后的小区名称 | 鑫苑名家一期 |
| rooms | int | 户型中的室数 | 2 |
| halls | int | 户型中的厅数 | 1 |
| area | decimal(8,2) | 建筑面积（㎡） | 62.46 |
| price | int | 月租金（元） | 1850 |
| price_per_sqm | decimal(8,2) | 平米单价（元/㎡） | 29.62 |
| is_abnormal | tinyint | 是否异常价格，1=异常，0=正常 | 0 |

说明：`clean_rent` 为 ETL 清洗后的标准化房源数据，主要用于后续分析与可视化。

## 表三：district_summary

| 字段名 | 类型 | 含义 | 示例值 |
| --- | --- | --- | --- |
| id | int | 自增主键 | 1 |
| district | varchar(50) | 行政区 | 锦江区 |
| business_area | varchar(100) | 商圈或业务片区 | 桑梓街 |
| avg_rent | decimal(10,2) | 平均租金（元/月） | 2330.50 |
| avg_price_per_sqm | decimal(10,2) | 平均平米单价（元/㎡） | 38.20 |
| avg_area | decimal(8,2) | 平均面积（㎡） | 58.75 |
| total_houses | int | 房源总数 | 45 |
| ai_comment | text | AI 生成的区域点评文本 | 该区域租房性价比较高，... |
| update_time | timestamp | 最近更新时间 | 2026-06-10 15:20:00 |

说明：`district_summary` 为汇总统计表，按区/商圈汇总房源指标并存储 AI 点评结果。

## 表关系

- `raw_rent.house_id` 与 `clean_rent.house_id` 关联：ETL 通过 `house_id` 判断原始房源是否已清洗，避免重复写入。
- `clean_rent.district` 与 `district_summary.district` 关联：`district_summary` 根据清洗后的行政区对房源进行汇总分析。

关系说明：

- 爬虫将数据写入 `raw_rent`
- ETL 读取 `raw_rent`，清洗后写入 `clean_rent`
- 汇总分析层根据 `clean_rent` 生成 `district_summary`
